title: Migration guide

Hive 4.x is a complete rewrite of Hive built on top of Isar. While the API remains similar, there are several breaking changes and new features you should be aware of when migrating from Hive 2.x.

<Warning>
Hive 4.x is not compatible with data created by Hive 2.x. You'll need to migrate your existing data manually.
</Warning>

## Major changes in Hive 4.x

From the CHANGELOG.md:

### New capabilities

- **Much more resource efficiency** - Built on Isar's native database engine
- **Support for access from multiple isolates** - Safe concurrent access
- **Support for transactions** - Atomic read and write operations
- **No more issues with concurrent access and corruption** - Robust locking
- **Vastly reduced startup time** - Faster box opening
- **No more code generation** - Use `.fromJson()` and `.toJson()` instead

### Breaking changes

1. **Type adapters replaced** - No more `TypeAdapter` classes with code generation
2. **Synchronous API** - All operations are now synchronous (no async `openBox`)
3. **Box initialization** - `Hive.box()` opens or returns existing box
4. **New registration system** - `Hive.registerAdapter()` API changed
5. **Transactions required** - Some operations must be wrapped in transactions

## Installation changes

### Old (Hive 2.x)

```yaml
dependencies:
  hive: ^2.2.3
  hive_flutter: ^1.1.0

dev_dependencies:
  hive_generator: ^2.0.0
  build_runner: ^2.3.0
```

### New (Hive 4.x)

```yaml
dependencies:
  hive: ^4.0.0
  isar_flutter_libs: ^4.0.0-dev.13
  path_provider: ^2.1.0

# No dev_dependencies needed!
```

<Note>
Hive 4.x eliminates code generation entirely. Remove `hive_generator` and `build_runner` from your project.
</Note>

## Initialization changes

### Old (Hive 2.x)

```dart
import 'package:hive_flutter/hive_flutter.dart';

void main() async {
  await Hive.initFlutter();
  await Hive.openBox('myBox');
  runApp(MyApp());
}
```

### New (Hive 4.x)

```dart
import 'package:hive/hive.dart';
import 'package:path_provider/path_provider.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final dir = await getApplicationDocumentsDirectory();
  Hive.defaultDirectory = dir.path;
  
  runApp(MyApp());
}
```

**Key differences:**
- No more `await Hive.initFlutter()`
- Set `Hive.defaultDirectory` instead of calling `init()`
- No need to `await openBox()` - boxes open synchronously with `Hive.box()`

## Box opening changes

### Old (Hive 2.x)

```dart
// Async opening
final box = await Hive.openBox('myBox');

// Get already-opened box
final box = Hive.box('myBox');

// Lazy box
final lazyBox = await Hive.openLazyBox('myLazyBox');
```

### New (Hive 4.x)

```dart
// Synchronous opening (opens if needed, returns if already open)
final box = Hive.box(name: 'myBox');

// No more lazy boxes - all boxes are efficient
final box = Hive.box(name: 'myBox');
```

**Key differences:**
- `Hive.box()` is synchronous and opens boxes on-demand
- No distinction between `openBox()` and `box()` - use `Hive.box()`
- Lazy boxes removed - Hive 4.x is efficient by default
- Use named parameter `name:` instead of positional argument

## Type adapter changes

This is the biggest change in Hive 4.x.

### Old (Hive 2.x)

```dart
// Define model with annotation
@HiveType(typeId: 0)
class Person extends HiveObject {
  @HiveField(0)
  String name;
  
  @HiveField(1)
  int age;
}

// Generate code with build_runner
// $ flutter pub run build_runner build

// Register adapter
void main() async {
  await Hive.initFlutter();
  Hive.registerAdapter(PersonAdapter());
  await Hive.openBox<Person>('persons');
  runApp(MyApp());
}
```

### New (Hive 4.x)

```dart
// Define model with fromJson/toJson
class Person {
  Person({required this.name, required this.age});

  factory Person.fromJson(Map<String, dynamic> json) => Person(
    name: json['name'] as String,
    age: json['age'] as int,
  );

  final String name;
  final int age;

  Map<String, dynamic> toJson() => {
    'name': name,
    'age': age,
  };
}

// Register adapter with fromJson function
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final dir = await getApplicationDocumentsDirectory();
  Hive.defaultDirectory = dir.path;
  
  Hive.registerAdapter('Person', Person.fromJson);
  
  runApp(MyApp());
}

// Use normally
final box = Hive.box<Person>(name: 'persons');
box.put('john', Person(name: 'John', age: 30));
```

**Key differences:**
- No more `@HiveType()` or `@HiveField()` annotations
- No more code generation
- Use `.fromJson()` and `.toJson()` instead of generated adapters
- Type name is a string (e.g., `'Person'`) instead of typeId number
- No need to extend `HiveObject`

<Tip>
Many modern code generators (like `json_serializable`, `freezed`) already generate `.fromJson()` and `.toJson()` methods, making migration easier.
</Tip>

## API changes

### Box operations

Most basic operations remain the same:

```dart
// ✅ Same in both versions
box.put('key', 'value');
final value = box.get('key');
box.delete('key');
box.clear();
box['key'] = 'value';
final value = box['key'];
```

Some operations have changed:

```dart
// Old (Hive 2.x)
await box.compact();  // Manual compaction
final isOpen = box.isOpen;
await box.close();

// New (Hive 4.x)
// No manual compaction needed - automatic
final isOpen = box.isOpen;
box.close();  // Synchronous
```

### Encryption

```dart
// Old (Hive 2.x)
import 'package:hive/hive.dart';
final encryptionKey = Hive.generateSecureKey();
final box = await Hive.openBox(
  'secureBox',
  encryptionCipher: HiveAesCipher(encryptionKey),
);

// New (Hive 4.x)
final box = Hive.box(
  name: 'secureBox',
  encryptionKey: 'your-secure-key-string',
);
```

**Key differences:**
- Pass `encryptionKey` directly as a string
- No need for `HiveAesCipher` wrapper
- Encryption key generation is now your responsibility

### Watch operations

```dart
// Old (Hive 2.x)
box.watch().listen((BoxEvent event) {
  print('Key: ${event.key}, Value: ${event.value}');
});

box.watch(key: 'specificKey').listen((BoxEvent event) {
  print('Value changed: ${event.value}');
});

// New (Hive 4.x)
box.watch().listen((_) {
  print('Box changed');
});

box.watchKey('specificKey').listen((value) {
  print('Value changed: $value');
});
```

**Key differences:**
- `watch()` now emits `void` instead of `BoxEvent`
- Use `watchKey(key)` instead of `watch(key: key)`
- `watchKey()` emits the actual value, not an event object

## Transaction support (new in 4.x)

Hive 4.x adds explicit transaction support:

```dart
// Hive 2.x - no explicit transactions
box.put('key1', 'value1');
box.put('key2', 'value2');
box.put('key3', 'value3');
// Each operation commits separately

// Hive 4.x - explicit transactions
box.write(() {
  box.put('key1', 'value1');
  box.put('key2', 'value2');
  box.put('key3', 'value3');
  // All committed atomically
});

// Read transactions
box.read(() {
  final value1 = box.get('key1');
  final value2 = box.get('key2');
  // Consistent snapshot
});
```

<Note>
In Hive 4.x, individual operations like `put()` and `delete()` automatically create transactions. Use explicit `write()` for batching multiple operations.
</Note>

## Multi-isolate support (new in 4.x)

Hive 4.x adds built-in support for multiple isolates:

```dart
// Hive 2.x - isolate access not supported
// (would cause corruption)

// Hive 4.x - safe multi-isolate access
final box = Hive.box();
box.put('data', 'main isolate');

await Hive.compute(() {
  final box = Hive.box();
  print(box.get('data')); // 'main isolate'
  box.put('data', 'background isolate');
});

print(box.get('data')); // 'background isolate'
```

## Data migration strategy

Since Hive 4.x uses a different storage format, you need to manually migrate data:

### Migration script

```dart
import 'package:hive/hive.dart' as hive4;
import 'package:hive/hive.dart' as hive2; // Old version

Future<void> migrateData() async {
  // 1. Open old Hive 2.x box
  await hive2.Hive.initFlutter();
  final oldBox = await hive2.Hive.openBox('myBox');
  
  // 2. Read all data from old box
  final allData = <String, dynamic>{};
  for (final key in oldBox.keys) {
    allData[key.toString()] = oldBox.get(key);
  }
  
  // 3. Initialize Hive 4.x
  final dir = await getApplicationDocumentsDirectory();
  hive4.Hive.defaultDirectory = dir.path;
  
  // 4. Create new Hive 4.x box
  final newBox = hive4.Hive.box(name: 'myBox');
  
  // 5. Write data to new box
  newBox.putAll(allData);
  
  // 6. Close and optionally delete old box
  await oldBox.close();
  await hive2.Hive.deleteBoxFromDisk('myBox');
  
  print('Migration complete: ${allData.length} items migrated');
}
```

### Gradual migration

For large apps, migrate boxes one at a time:

```dart
class MigrationManager {
  static const _prefsKey = 'hive_migration_status';
  
  static Future<void> migrateIfNeeded() async {
    final prefs = await SharedPreferences.getInstance();
    final migrated = prefs.getBool(_prefsKey) ?? false;
    
    if (!migrated) {
      await _migrateBox('settings');
      await _migrateBox('cache');
      await _migrateBox('userdata');
      
      await prefs.setBool(_prefsKey, true);
      print('All boxes migrated successfully');
    }
  }
  
  static Future<void> _migrateBox(String boxName) async {
    // Migration logic for single box
    print('Migrating $boxName...');
    // ... migration code ...
  }
}

// In main()
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await MigrationManager.migrateIfNeeded();
  
  // Now use Hive 4.x normally
  final dir = await getApplicationDocumentsDirectory();
  Hive.defaultDirectory = dir.path;
  
  runApp(MyApp());
}
```

<Warning>
Back up user data before migration. Consider offering users the option to export their data before updating.
</Warning>

## Testing changes

```dart
// Old (Hive 2.x)
setUp(() async {
  await Hive.initFlutter();
  await Hive.openBox('testBox');
});

tearDown(() async {
  await Hive.deleteBoxFromDisk('testBox');
});

// New (Hive 4.x)
setUp(() {
  Hive.defaultDirectory = Directory.systemTemp.path;
  // Box opens automatically on first access
});

tearDown(() {
  final box = Hive.box(name: 'testBox');
  box.deleteFromDisk();
  Hive.closeAllBoxes();
});
```

## Common migration issues

### Issue: "No directory specified"

```dart
// ❌ Error
final box = Hive.box();
// HiveError: No directory specified and no default directory set

// ✅ Fix
Hive.defaultDirectory = '/path/to/directory';
final box = Hive.box();
```

### Issue: "Type adapter not registered"

```dart
// ❌ Error
box.put('person', Person(name: 'Alice', age: 30));
// HiveError: No adapter registered for type Person

// ✅ Fix
Hive.registerAdapter('Person', Person.fromJson);
box.put('person', Person(name: 'Alice', age: 30));
```

### Issue: Missing .fromJson/.toJson

```dart
// ❌ Won't work
class Person {
  String name;
  int age;
}

// ✅ Add serialization methods
class Person {
  Person({required this.name, required this.age});
  
  factory Person.fromJson(Map<String, dynamic> json) => Person(
    name: json['name'],
    age: json['age'],
  );
  
  final String name;
  final int age;
  
  Map<String, dynamic> toJson() => {'name': name, 'age': age};
}
```

## Feature comparison

| Feature | Hive 2.x | Hive 4.x |
|---------|----------|----------|
| Code generation | Required | Not needed |
| Async box opening | Yes | No (synchronous) |
| Multi-isolate | No | Yes |
| Transactions | Implicit | Explicit |
| Lazy boxes | Yes | No (all efficient) |
| Type adapters | TypeAdapter classes | fromJson functions |
| Encryption | HiveAesCipher | String key |
| Box size limit | None | maxSizeMiB parameter |
| Watches | BoxEvent objects | Direct values |

## Migration checklist

- [ ] Update dependencies in `pubspec.yaml`
- [ ] Remove `hive_generator` and `build_runner`
- [ ] Add `isar_flutter_libs` dependency
- [ ] Replace `Hive.initFlutter()` with `Hive.defaultDirectory`
- [ ] Change all `await Hive.openBox()` to `Hive.box()`
- [ ] Convert `@HiveType` classes to `.fromJson()/.toJson()`
- [ ] Update `registerAdapter()` calls
- [ ] Replace `HiveAesCipher` with direct encryption keys
- [ ] Update `watch()` listeners to handle new API
- [ ] Add explicit transactions where needed
- [ ] Implement data migration script
- [ ] Test thoroughly on development devices
- [ ] Back up user data before production release
- [ ] Update documentation and user guides
