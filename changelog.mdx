## Version 4.0.0

<Warning>
This version of Hive is currently in development and not ready for production use.
</Warning>

Hive 4.0.0 is a complete rewrite that brings significant improvements and changes. The new version uses Isar internally, bringing all the benefits of a native database to Hive.

### Key features

- **Much more resource efficiency** - Optimized memory and CPU usage
- **Support for access from multiple isolates** - True concurrent access without conflicts
- **Support for transactions** - Atomic operations with automatic rollback on errors
- **No more issues with concurrent access and corruption** - Robust multi-isolate support
- **Vastly reduced startup time** - Faster box opening and initialization
- **No more code generation** - Simpler setup with `fromJson`/`toJson` pattern

### Breaking changes

<Warning>
Hive 4.0.0 is not compatible with older versions. Migration is required for existing projects.
</Warning>

#### Box opening API

The box opening API has changed:

```dart
// Old (Hive 3.x)
final box = await Hive.openBox('myBox');

// New (Hive 4.0.0)
final box = Hive.box(name: 'myBox');
```

#### Type adapters

Custom type adapters no longer require code generation:

```dart
// Old (Hive 3.x)
@HiveType(typeId: 0)
class Person extends HiveObject {
  @HiveField(0)
  String name;
}

// New (Hive 4.0.0)
class Person {
  Person({required this.name});
  
  factory Person.fromJson(Map<String, dynamic> json) => Person(
    name: json['name'] as String,
  );
  
  final String name;
  
  Map<String, dynamic> toJson() => {'name': name};
}

Hive.registerAdapter('Person', Person.fromJson);
```

#### Synchronous operations

All operations are now synchronous by default:

```dart
// Old (Hive 3.x)
await box.put('key', 'value');

// New (Hive 4.0.0)
box.put('key', 'value');
```

For isolate-based operations, use `Hive.compute()`:

```dart
await Hive.compute(() {
  final box = Hive.box();
  box.put('key', 'value');
});
```

### Migration guide

<Note>
For detailed migration instructions, see the [migration guide](/advanced/migration).
</Note>

1. **Export your data** from Hive 3.x boxes before upgrading
2. **Update dependencies** to Hive 4.0.0 and remove code generation packages
3. **Update box opening calls** to use the new synchronous API
4. **Convert type adapters** to use `fromJson`/`toJson` pattern
5. **Import your data** into new Hive 4.0.0 boxes
